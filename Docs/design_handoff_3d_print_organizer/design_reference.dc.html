3D Print File Organizer — Concept Directions

Palette: warm near-black workshop tones with a single hot-orange accent (matches your "warm maker" pick, dark/dense pro-tool direction). Type is IBM Plex Sans for UI, IBM Plex Mono for technical readouts (dimensions, print time, layer height) — leaning into the precision-tool feel without going full CAD-blueprint.

Since a file can live in many nested folders at once, I've treated "folders" and "collections" as the same underlying tree (like Lightroom's Collections, but hierarchical) rather than two separate systems — a file just gets checked into however many tree nodes apply. See 3a for how multi-assign works.

3D thumbnails/renders below are placeholders (striped blocks) — real renders would come from actual 3MF/STL parsing. t1 below explores three library layouts; t2–t4 flesh out detail view, folder org, and batch tagging using 1a as the base.
